**gen-circleci-orb** turns a Rust [clap](https://docs.rs/clap) CLI into a CircleCI orb. It reads
the `--help` output of a clap-based binary and generates the complete unpacked orb — one command
and one job per subcommand, an executor, and a Dockerfile — plus, optionally, the CI
configuration that regenerates and republishes the orb automatically on every release.

The workflow is captured once in a `gen-circleci-orb.toml` file so it never has to be re-typed.
`init` scaffolds that file and patches your `.circleci/` config; `generate` (re)writes the orb
source from the binary's current `--help`; and `update` re-syncs the managed CI blocks as the
generator itself evolves — leaving your own jobs untouched. It can also wire in
[gen-orb-mcp](@/projects/gen-orb-mcp/index.md) so an AI assistant understands the generated orb.

This project dogfoods its own orb: gen-circleci-orb's CI is generated by gen-circleci-orb.
